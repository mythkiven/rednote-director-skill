# 24 种图像提示词模板

使用方式：复制对应风格模板，把 `[主题]`、`[标题]`、`[页面任务]`、`[核心元素]` 替换为当前内容。默认建议后期添加中文文字；如果模型中文能力弱，请要求“只生成可排版背景和文字占位”。

## 通用负面提示词

```text
不要廉价蓝紫渐变，不要随机霓虹线条，不要文字变形，不要小字堆积，不要 PPT bullet 列表，不要塑料质感，不要儿童卡通，不要低清晰度，不要过度装饰，不要元素遮挡标题，不要页码喧宾夺主。
```

## 1. 深色科技杂志风

- 适合：AI、Vibe Coding、Agent、技术趋势、方法论。
- 不适合：生活种草、温柔情绪、低信息密度内容。
- 视觉关键词：深色杂志、细网格、玻璃面板、数据流、银白文字、少量荧光绿。
- 版式结构：左上大标题，右下主视觉，中部结构线，底部小注释。

```text
生成一张 3:4 竖版小红书图文页面，主题是「[主题]」，页面任务是「[页面任务]」。使用深色科技杂志风：黑色和深灰背景，极细网格和结构线，右下角有抽象 Agent 网络 / 代码窗口 / 数据流作为主视觉，左上保留大标题安全区，标题使用粗黑体无衬线，正文使用小号 Inter 风格。色彩以银白和灰色为主，只用少量荧光绿强调关键词、编号或连线。画面要像高端 AI 产品发布杂志封面，留白清晰，信息层级明确。不要生成真实中文正文，只保留清晰文字占位区。
负面：廉价蓝紫渐变、随机霓虹、文字变形、复杂无意义背景、字号过小、PPT 列表。
```

## 2. 黑白灰 + 荧光绿冲击风

- 适合：强观点、反常识、技术布道、爆款封面。
- 不适合：柔和产品页、长教程内页。
- 视觉关键词：巨大标题、强留白、黑白灰、荧光绿关键词。
- 版式结构：标题占 40%，一个绿色关键词做视觉锚点。

```text
生成一张 3:4 竖版小红书封面，主题是「[主题]」。黑白灰极简冲击风，背景为纯黑或近黑，标题区占画面 40%，使用超大粗黑体，标题中只有一个关键词用荧光绿突出。画面只保留少量细线、编号和几何切割，形成犀利、克制、有攻击性的视觉。正文区域很少，底部放一行小注释占位。整体要有强对比和高级留白，不要花哨。
负面：多色渐变、荧光绿铺满、装饰过多、表情包感、文字拥挤。
```

## 3. Notion 高级卡片风

- 适合：工具推荐、工作流、方法清单、知识管理。
- 不适合：争议封面、宏大商业叙事。
- 视觉关键词：知识库、卡片、标签、目录、折叠块。
- 版式结构：浅灰背景，多张白色卡片，标题区清晰。

```text
生成一张 3:4 竖版小红书知识卡片页，主题是「[主题]」。浅灰或米白背景，多张低圆角白色卡片有秩序排列，包含标签、步骤、引用块、checkbox、目录感结构。顶部保留清晰大标题区，中部用 3-5 张卡片承载重点，底部有小号注释区。字体使用干净无衬线，黑字和灰字形成层级，少量蓝灰或薄荷绿强调。整体像高级 Notion 页面但更适合手机阅读。
负面：卡片过多、字号过小、贴纸感、过度阴影、PPT 感。
```

## 4. 液态玻璃 / 弥散极光风

- 适合：产品发布、个人品牌、未来感封面。
- 不适合：高密度教程、严肃报告。
- 视觉关键词：深色、低饱和弥散光、玻璃面板、柔和未来感。
- 版式结构：背景光晕，前景玻璃卡片，标题在高对比区域。

```text
生成一张 3:4 竖版小红书视觉封面，主题是「[主题]」。深色背景，低饱和蓝绿 / 粉橙弥散极光从边缘柔和扩散，前景有一块半透明液态玻璃面板，标题区放在清晰暗色区域，保留 40% 以上留白。字体为现代无衬线，主标题大而克制，辅助信息用小标签。光效要柔和高级，不能盖住文字区域。
负面：彩虹渐变铺满、强紫蓝俗气光效、文字被光效干扰、塑料质感。
```

## 5. 极简产品发布会风

- 适合：产品展示、工具介绍、MVP 发布。
- 不适合：复杂教程、情绪独白。
- 视觉关键词：Apple-like、产品主视觉、hero section、单一强调色。
- 版式结构：中心产品，顶部或左侧大标题，少量卖点标签。

```text
生成一张 3:4 竖版小红书产品发布页，主题是「[主题]」。中心展示一个软件界面 / 产品 mockup / 工具主视觉，背景为黑白灰极简发布会风，使用单一强调色点亮关键按钮或标签。标题短而大，放在顶部或左侧清晰安全区，卖点用 2-3 个小标签围绕产品。整体像高端官网 hero section，克制、有光影、无杂乱装饰。
负面：PPT bullet、廉价样机、强反光、过多卖点、促销海报感。
```

## 6. 反差冲击封面风

- 适合：反常识观点、争议选题、错误 vs 正确。
- 不适合：温柔品牌页、长期主义内容首图。
- 视觉关键词：左右对比、错误 / 正确、强标题、警示色。
- 版式结构：左右或上下对照，大标题压住视觉中心。

```text
生成一张 3:4 竖版小红书强冲突封面，主题是「[主题]」。画面采用左右对比结构，左侧代表错误旧方式，用低饱和红和暗色阴影；右侧代表正确新方式，用低饱和绿和清晰结构。中央或顶部放超大标题「[标题]」的文字占位，形成一眼停住的反差。使用粗黑体、强分割线、少量对勾和叉号图形，整体高级克制，不要土味营销。
负面：红绿刺眼、巨大感叹号、表情包风、低俗营销、文字太多。
```

## 7. 架构图 / 系统拆解风

- 适合：Agent 架构、工程系统、复杂方法论。
- 不适合：低信息密度情绪文。
- 视觉关键词：模块节点、箭头、输入处理输出、系统层级。
- 版式结构：顶部结论，中部架构，底部输出。

```text
生成一张 3:4 竖版小红书系统拆解页，主题是「[主题]」。深色背景，顶部是一句清晰结论的标题占位，中部是模块化架构图：输入、处理、工具、数据、输出五类节点，用绿色或电光蓝细线连接。节点使用白色或深灰玻璃卡片，层级清楚，箭头方向明确。底部放结果区和小注释。整体专业、干净、像高级产品架构图。
负面：箭头混乱、节点过多、文字过小、伪 3D 图标、PPT 流程图感。
```

## 8. 手机截图改造风

- 适合：App 工作流、桌面整理、真实案例。
- 不适合：没有截图的抽象观点。
- 视觉关键词：真实截图、标注线、编号、功能标签。
- 版式结构：中间手机截图，周围解释标注。

```text
生成一张 3:4 竖版小红书手机截图讲解页，主题是「[主题]」。中间放一张真实比例的手机截图占位，不能拉伸变形；四周留出标注区，用细线、编号和短标签解释功能区。背景使用低调深灰或米白，标题在顶部清晰显示，字体为现代无衬线。整体真实、可模仿、有高级感，标注线克制。
负面：截图拉伸、夸张手机边框、贴纸感、标注太密、文字太小。
```

- 不适合：个人情绪、轻娱乐。
- 视觉关键词：商务、信任、数据卡片、行业场景。
- 版式结构：左标题，右数据 / 场景 / 流程。

```text
生成一张 3:4 竖版小红书商业提案页，主题是「[主题]」。深蓝黑背景，左侧放大标题和 2 个商业关键词，右侧使用数据卡片、行业场景图或流程图构成主视觉。配色为深蓝黑、暖白、银灰，少量暖金细线增强信任感。字体稳重清晰，标题有商业发布会气质。页面要专业可信，不像土味招商。
负面：土味招商海报、金色过度、股票软件感、廉价商务模板。
```


## 11. 高级白底杂志风

- 适合：认知文章、审美分享、个人观点。
- 不适合：强科技实验、黑客档案。
- 视觉关键词：白底、专栏、留白、细线、编辑感。
- 版式结构：大标题 + 小注释 + 细线分割。

```text
生成一张 3:4 竖版小红书白底杂志页，主题是「[主题]」。米白或纯白背景，大量留白，左上或中部放大标题占位，右下放一个极简图形或引用注释。使用黑、灰、少量红色或蓝色强调，细线组织版面。字体为高级无衬线，标题有字重对比，整体像杂志专栏，不像 PPT 标题页。
负面：空得没有重点、字体太细、装饰太多、PPT 感。
```

## 12. 红绿对错对比风

- 适合：Prompt 优化、设计改版、错误示范。
- 不适合：情绪文、品牌宣言。
- 视觉关键词：Bad / Good、Before / After、低饱和红绿。
- 版式结构：左右对比，顶部结论。

```text
生成一张 3:4 竖版小红书对错对比页，主题是「[主题]」。顶部放一句结论标题占位，下面左右两栏对比：左侧错误示范使用低饱和红和暗色卡片，右侧正确示范使用低饱和绿和清晰卡片。中间用细分隔线，卡片内保留短句占位。整体实用、清晰、有收藏价值。
负面：红绿刺眼、考试卷感、文字密集、卡片拥挤。
```

- 版式结构：档案标题 + 终端窗口 + 状态条。

```text

负面：低质黑客电影海报、乱码文字、过度故障效果、绿色铺满。
```

## 23. 高级电商详情页风
