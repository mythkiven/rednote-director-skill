# 07. 架构图 / 系统拆解风

## 风格定义

- 适合内容：Agent 架构、产品系统、API Contract、工程化、复杂方法论。
- 不适合内容：低信息密度情绪文、纯审美分享。
- 视觉气质：专业、清晰、系统性强。
- 配色：深色底 / 白色卡片 / 绿色或蓝色连线。
- 字体：标题粗，节点标签中等，注释小而清晰。
- 构图：模块化节点、箭头、层级结构、输入 / 处理 / 输出。
- 常用元素：节点、连接线、流程箭头、状态灯、分层面板。
- 图像提示词模板：3:4 竖版系统架构图，顶部一句结论，中部模块节点和箭头，底部输出结果区，深色背景，白色卡片，绿色连线。
- 负面提示词：箭头混乱、节点过多、文字过小、伪 3D 图标。
- 示例标题类型：“一个 Agent 应用到底由什么组成？”。

## 图像提示词模板

- 适合：Agent 架构、工程系统、复杂方法论。
- 不适合：低信息密度情绪文。
- 视觉关键词：模块节点、箭头、输入处理输出、系统层级。
- 版式结构：顶部结论，中部架构，底部输出。

```text
生成一张 3:4 竖版小红书系统拆解页，主题是「[主题]」。深色背景，顶部是一句清晰结论的标题占位，中部是模块化架构图：输入、处理、工具、数据、输出五类节点，用绿色或电光蓝细线连接。节点使用白色或深灰玻璃卡片，层级清楚，箭头方向明确。底部放结果区和小注释。整体专业、干净、像高级产品架构图。
负面：箭头混乱、节点过多、文字过小、伪 3D 图标、PPT 流程图感。
```
