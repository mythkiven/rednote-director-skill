# 17. 软件界面 UI 风

## 风格定义

- 适合内容：AI 产品、SaaS、工具、工作台、自动化流程。
- 不适合内容：没有产品界面支撑的情绪文。
- 视觉气质：产品感、真实、可落地、专业。
- 配色：深色 UI 或浅色 UI，强调层级真实。
- 字体：界面字体清晰，数字和状态标签精确。
- 构图：Dashboard、侧边栏、卡片、操作面板、数据面板。
- 常用元素：导航栏、状态栏、图表、任务卡片、Agent 卡片。
- 图像提示词模板：3:4 竖版软件界面展示页，真实 dashboard 布局，左侧侧边栏，中间任务卡片，右侧状态面板，标题区域与界面分层明确。
- 负面提示词：伪 UI、按钮乱码、过度玻璃、界面细节不可读。
- 示例标题类型：“我给自己搭了一个 AI 操作系统”。

## 图像提示词模板

- 适合：AI 产品、SaaS、工具、工作台。
- 不适合：无产品界面支撑的情绪文。
- 视觉关键词：Dashboard、侧边栏、任务卡、状态面板。
- 版式结构：真实 UI 界面 + 标题区。

```text
生成一张 3:4 竖版小红书软件界面展示页，主题是「[主题]」。画面主体是一个真实可信的 dashboard：左侧侧边栏，中间任务卡片，右侧 Agent 状态面板或数据面板。顶部保留标题安全区，界面使用深色或浅色现代 UI，卡片层级清楚，按钮和状态标签整齐。整体产品感强、可落地。
负面：伪 UI、乱码按钮、界面过密、过度玻璃、细节不可读。
```
