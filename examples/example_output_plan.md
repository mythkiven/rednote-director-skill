# 示例输出方案

主题：为什么普通人现在必须学习 Vibe Coding？

# 选题判断

- 输入类型：选题。
- 内容类型：AI 观点 + 方法论 + 普通人生产力教育。
- 目标受众：想用 AI 提升效率，但不会编程、也不知道如何与 AI 协作的人。
- 内容机会：把“学编程”的焦虑，转成“学会指挥 AI 做产品”的新认知。

# 核心观点

- 核心观点一句话：普通人学习 Vibe Coding，不是为了成为程序员，而是为了拥有把想法变成产品的 AI 协作能力。
- 传播钩子：未来的差距，不是会不会写代码，而是会不会把任务交给 AI。
- 收藏理由：读者可以收藏这套“想法 -> 需求 -> 生成 -> 验收”的流程。
- 评论区话题：你最想用 AI 做一个什么小工具？

# 风格判断报告

## 内容类型判断

- 当前内容属于：观点型 + 方法论型 + 科普型。
- 判断依据：
  - 主题不是单纯教学，而是在改变读者对“编程门槛”的认知。
  - 需要解释 Vibe Coding 的价值，并给出可执行流程。
  - 读者既需要被封面击中，也需要在内页获得收藏价值。

## 传播目标判断

- 主要目标：吸引点击 + 建立专业度。
- 次要目标：促成收藏 + 引发评论。
- 不适合的目标：直接转化；这篇更适合做认知入口。

## 读者情绪判断

- 封面：被击中，意识到“我不能再只旁观 AI”。
- 第 2-3 页：焦虑被解释，知道自己不是不会编程，而是缺少协作方法。
- 第 4-6 页：想收藏，获得可复制流程。
- 结尾页：被鼓励，愿意评论自己的想法。

## 信息密度判断

- 当前信息密度：高。
- 推荐版式：封面大标题型 + 架构图型 + 卡片拆解型 + 对比表格型。
- 注意事项：必须拆页，不要把 Vibe Coding 的定义、原因、流程和案例塞在一页。

## 风格匹配结果

- 主风格：深色科技杂志风。
- 辅助风格：黑白灰 + 荧光绿冲击风；架构图 / 系统拆解风；Notion 高级卡片风。
- 不建议使用的风格：液态玻璃 / 弥散极光风作为主风格。
- 选择理由：主题需要专业、理性和冲击力；过度梦幻会削弱方法论可信度。
- 风格关键词：深色、网格、荧光绿关键词、Agent 节点、结构卡片。
- 配色建议：黑、深灰、银白，荧光绿只用于关键词和连线。
- 字体建议：中文粗黑体 + HarmonyOS Sans；英文 Inter。
- 构图建议：封面大标题，内页用节点、卡片和对比结构递进。

# 推荐风格方案

## 方案 A：点击优先

- 风格组合：反差冲击封面风 + 黑白灰荧光绿冲击风。
- 适合原因：更容易让用户停下来。
- 风险：如果内页不够系统，容易变成情绪观点。

## 方案 B：收藏优先

- 风格组合：Notion 高级卡片风 + 架构图 / 系统拆解风。
- 适合原因：方法论清晰，适合收藏复用。
- 风险：封面点击力可能偏弱。

## 方案 C：高级感 / 个人品牌优先

- 风格组合：深色科技杂志风 + 个人品牌宣言风。
- 适合原因：建立账号的专业审美和技术观点识别度。
- 风险：需要标题足够尖锐，否则容易显得冷。

我的推荐：
选择方案 A + B 的组合。

推荐理由：

- 封面必须先用强观点拦截用户。
- 内页必须用架构图和卡片建立收藏价值。
- 深色科技杂志风能承接 AI / Agent 主题。
- 荧光绿只做关键词强调，可以避免廉价科技感。

# 统一视觉规范

- 画幅：3:4 竖版。
- 主色：黑、深灰、银白。
- 强调色：荧光绿，只用于关键词、节点连线、状态标签。
- 字体：中文 HarmonyOS Sans / 思源黑体风格；英文 Inter。
- 标题层级：封面标题最大，内页标题保持统一字号区间。
- 卡片样式：低圆角，轻阴影或玻璃感，避免圆润可爱。
- 页码 / 角标：小号，放底部边缘，不抢主题。
- 留白规则：每页至少 25% 清晰留白；封面至少 35% 标题安全区。
