# 高质量图像生成提示词示例

主题：为什么普通人现在必须学习 Vibe Coding？

## 1. 封面提示词

```text
生成一张 3:4 竖版小红书封面，主题是「为什么普通人现在必须学习 Vibe Coding」。使用黑白灰 + 荧光绿冲击风结合深色科技杂志风：近黑背景，左侧超大标题安全区，标题占画面 40%，关键词「Vibe Coding」用荧光绿突出；右侧是抽象的人与 AI 协作界面，由代码窗口、任务卡片和 Agent 节点构成。背景只有极细网格和少量结构线，底部有小号英文注释占位。画面要像高级 AI 杂志封面，强对比、克制、手机端一眼可读。不要生成真实中文正文，只保留清晰文字占位。
负面：廉价蓝紫渐变、随机霓虹线条、复杂无意义背景、PPT 列表、文字变形、字号过小。
```

## 2. 问题页提示词

```text
生成一张 3:4 竖版小红书问题页，主题是「不是不会写代码，而是不会指挥 AI」。使用反差冲击封面风：左右对比结构，左侧是普通人把需求直接丢给 AI 的混乱状态，暗红低饱和；右侧是把任务拆解成模块、约束、验收标准的清晰状态，低饱和绿。顶部保留一句大标题安全区，中部用两块卡片对比错误方式和正确方式。字体为粗黑体和 Inter 小注释，整体高级克制。
负面：红绿刺眼、表情包、土味营销、大量小字、夸张感叹号。
```

## 3. 架构页提示词

```text
生成一张 3:4 竖版小红书系统架构页，主题是「Vibe Coding 的真正流程」。深色背景，顶部一句结论标题占位，中部是清晰流程架构图：想法输入 -> 需求拆解 -> 约束说明 -> AI 生成 -> 人类验收 -> 迭代修正。每个节点是深灰玻璃卡片，使用荧光绿细线连接，底部有输出结果区。画面专业、清晰、像高级产品系统图，手机端能读懂结构。
负面：箭头混乱、节点过多、文字太小、PPT 流程图感、伪 3D 图标。
```

## 4. 案例页提示词

```text
生成一张 3:4 竖版小红书案例拆解页，主题是「一个页面需求如何交给 AI」。使用 Notion 高级卡片风 + 红绿对错对比风：浅灰背景，顶部大标题，中部左右两栏卡片。左侧卡片标题为错误示范占位，呈现模糊需求；右侧卡片标题为更好写法占位，拆成目标、模块、风格、验收四块。低饱和红和低饱和绿只用于标签，不铺满背景。整体可收藏、清晰、有教程价值。
负面：卡片拥挤、文字过小、红绿刺眼、贴纸感、PPT 感。
```

## 5. 总结页提示词

```text
生成一张 3:4 竖版小红书总结页，主题是「未来会写代码的人，不一定最会做产品」。使用个人品牌宣言风 + 黑白灰荧光绿冲击风：黑白灰背景，大面积留白，中间偏左放一句大宣言文字占位，荧光绿只强调一个关键词「指挥 AI」。右下角放极简个人标签和下一篇预告区域。整体克制、锋利、有个人观点感，像账号系列的结尾签名页。
负面：鸡汤海报、字体花哨、装饰过多、巨大页码、文字不可读。
```
