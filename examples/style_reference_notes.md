# 风格判断参考记录

## 示例 1

输入主题：

“AI 写代码总翻车？因为你没给它眼睛”

风格判断：

- 内容类型：方法论 + 反常识观点
- 主目标：点击 + 收藏
- 推荐风格：
  - 封面：反差冲击封面风
  - 内页：架构图 / 系统拆解风
  - 案例页：红绿对错对比风
  - 结尾：黑白灰 + 荧光绿冲击风

判断理由：

- 标题本身有冲突，需要封面先把“翻车原因”打出来。
- 内页要解释“眼睛”具体是什么，所以需要系统拆解。
- 案例页适合用错误 / 正确对比，增强收藏价值。

## 示例 2

输入主题：

“我把手机桌面整理成了一个信息仪表盘”

风格判断：

- 内容类型：真实案例 + 方法教程
- 主目标：收藏 + 模仿
- 推荐风格：
  - 封面：手机截图改造风
  - 内页：Notion 高级卡片风
  - 说明页：截图标注风
  - 结尾：高级工具清单风

判断理由：

- 用户需要看到真实桌面才会相信。
- 说明页必须保留截图比例，用标注降低学习成本。
- 结尾给工具清单，方便收藏和复用。

## 示例 3

输入主题：

“义乌商户为什么必须看懂 AI”

风格判断：

- 内容类型：商业趋势 + 产业方案
- 主目标：建立专业度 + 转化
- 推荐风格：
  - 封面：高级商业提案风
  - 背景：全球贸易网络风
  - 数据页：数据报告 / 趋势洞察风
  - 方案页：架构图 / 系统拆解风

判断理由：

- 主题面向商户和产业，需要可信度而不是网感。
- 全球贸易网络能表达义乌和出海关系。
- 数据页建立趋势判断，方案页承接商业转化。

完整规划参考：`examples/example_output_yiwu_plan.md`

## 示例 4

输入主题：

“我用 Codex 做了一个自己的 Codex”

风格判断：

- 内容类型：实验复盘 + 技术故事
- 主目标：好奇心 + 专业度
- 推荐风格：
  - 封面：赛博档案 / 黑客文件风
  - 内页：软件界面 UI 风
  - 过程页：未来实验室风
  - 总结页：个人品牌宣言风

判断理由：

- 标题有技术实验感，适合档案式封面。
- 过程页要展示系统如何工作，不能只写故事。
- 结尾页适合回到个人品牌，形成“公开构建”的识别度。
