# 页面结构规则

## 基础原则

- 默认 3:4 竖版。
- 默认 8 页；用户指定页数时按用户要求。
- 一页只承担一个传播任务。
- 标题必须先被看见，正文必须服务标题。
- 手机端优先，正文尽量控制在 2-4 行。
- 页码、角标、装饰线不能抢过标题。
- 页面不能像普通 PPT 截图。

## 8 页默认结构

| 页码 | 页面任务 | 设计重点 |
| --- | --- | --- |
| 01 | 封面 | 强钩子、大标题、主视觉、清晰情绪 |
| 02 | 痛点 | 指出用户正在犯的错或忽略的问题 |
| 03 | 认知 | 解释为什么重要，建立判断框架 |
| 04 | 方法 | 给出结构、模型、步骤或系统 |
| 05 | 案例 | 展示具体例子，降低理解成本 |
| 06 | 操作 | 给可执行步骤，增强收藏价值 |
| 07 | 总结 | 收束观点，形成可复述金句 |
| 08 | 引导 | 收藏、评论、关注、下一篇预告 |

## 封面规则

- 标题占画面 30%-45%。
- 一屏只强调一个钩子，不要同时塞满卖点。
- 主视觉要和标题有关系，不能只是装饰。
- 要有清晰文字安全区，避免光效、纹理、图形干扰标题。
- 封面可以更冲击，但不能牺牲可读性。

## 内页规则

- 每页一个核心标题。
- 每页一个主要信息结构：对比、流程、架构、清单、案例、图表、标注。
- 内页之间要有节奏变化：不能 8 页都是中心大字，也不能 8 页都是卡片列表。
- 色彩、字体、角标和辅助线必须统一。
- 高信息密度内容必须拆页，不允许压缩字号硬塞。

## 结尾页规则

- 收束一句能被用户记住的话。
- 给明确行动：收藏、评论关键词、关注下一篇。
- 结尾视觉可以更简洁，像品牌签名。
- 不要用廉价二维码感或大促销感。
