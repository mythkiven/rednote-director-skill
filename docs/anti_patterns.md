# 反模式文档

## 1. 廉价 AI 科技风

表现：

- 蓝紫渐变。
- 随机发光线条。
- 复杂但无意义的背景。
- 字很小。
- 像模板网站。

避免方式：

- 控制色彩数量。
- 重点放在标题和结构。
- 用真实信息层级替代装饰。
- 只保留少量有意义的科技线条。
- 背景必须让位于标题和主视觉。

## 2. PPT 感太重

表现：

- 每页像演示文稿截图。
- 页面中心全是 bullet points。
- 缺少社交媒体冲击力。

避免方式：

- 强化封面钩子。
- 采用杂志构图。
- 使用视觉主元素。
- 让每页只承担一个任务。
- 内页使用卡片、结构图、对比图、标注图，而不是普通列表页。

## 3. 信息过载

表现：

- 每页都想讲完。
- 文字太多。
- 用户不想读。

避免方式：

- 一页只讲一个点。
- 大标题先行。
- 复杂内容拆成结构图。
- 信息密度高时增加页数，而不是压缩字号。

## 4. 伪高级感

表现：

- 只会写“高级、极简、科技”。
- 没有版式细节。
- 看起来像空洞风格词堆叠。

避免方式：

- 必须描述留白、层级、字体、材质、主次关系。
- 必须说明标题、主视觉、辅助信息的布局比例。
- 必须说明高亮色只用在哪里。

## 5. 文字不可读

表现：

- 字太小。
- 文字和背景对比不足。
- 文字被光效、纹理、图形干扰。

避免方式：

- 标题必须有清晰背景区域。
- 正文必须控制在 2-4 行。
- 重要信息必须放在高对比区域。
- 提示词中明确要求“文字安全区”和“高对比标题区”。

## 6. 风格不统一

表现：

- 每页像不同模板拼接。
- 配色变化过大。
- 字体层级混乱。

避免方式：

- 统一字体系统。
- 统一主色和强调色。
- 统一页码、角标、标签和辅助线风格。
- 允许构图变化，但不允许视觉语言断裂。
